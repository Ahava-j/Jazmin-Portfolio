Counting Inversions

Project 1

This project involved implementing an inversion count algorithm in C++ to practice designing efficient solutions for problems involving array comparisons. An inversion is a pair of elements in an array where the earlier element is greater than the later one (i.e., arr[i] > arr[j] for i < j). Counting these inversions is useful in various domains like measuring how far a sequence is from being sorted valuable in sorting analysis, competitive programming, and data preprocessing tasks. The main purpose of the project was to deepen my understanding of divide-and-conquer algorithms and improve my ability to write time efficient code. It was important to me because I wanted to strengthen my foundation in algorithmic thinking which is an essential skill for technical interviews, programming competitions, and advanced software development. I used C++ and the Standard Template Library (STL) for vectors and basic I/O, and I worked in Visual Studio Code using the G++ compiler. I learned how to implement an optimized inversion count using a modified merge sort approach with O(n log n) complexity, compared to the brute-force O(nÂ²) method. I also improved my understanding of recursion and helper functions, and gained experience handling edge cases like empty arrays or duplicate values. Although this was an individual project, I discussed ideas with peers and learned how to clearly explain my logic, as well as the value of code reviews in catching logic errors. There are no known bugs at this time. The algorithm runs as expected and handles standard test cases correctly. Potential improvements include refactoring the code for better modularity, adding unit tests with a C++ testing framework like Catch2, creating a GUI or visual representation of inversion pairs, and optimizing memory usage for large datasets.
